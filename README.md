# ForCodingTest

기업 코딩 테스트를 준비하기 위한 레포지토리입니다.

매주 토요일과 일요일, 하루에 한 문제씩 Issue를 등록할 예정입니다.  
참여자는 가능하면 당일에 해당 문제를 해결한 뒤, 관련 Issue를 태그하여 PR을 올려 주세요.

이렇게 Issue 를 통해 관리를 하면 문제 목록을 따로 관리할 필요도 없어지고  
추후 해당 문제에 대한 다른 팀원들의 풀이를 모아볼 수 있어 편리합니다.
  
새롭게 참가하시는 분들은 TWL Repo와 동일한 방식으로 본인 폴더를 생성한 뒤, 그 안에서 파일을 관리해 주시면 됩니다.

이곳은 `TIL`이 아니라 `리뷰`를 위한 공간입니다.  
혼자만 보는 코드가 아니라 함께 읽고 피드백하는 코드라는 점을 고려하여, 다른 사람이 이해할 수 있도록 주석과 설명을 함께 작성해 주세요.

다른 사람이 이해하기 어려운 코드에는 충분한 리뷰를 드리기 어렵고, 결국 본인의 학습 기회를 줄이는 결과로 이어질 수 있습니다.

추천 문제, 개선 아이디어, 기타 제안 사항이 있다면 언제든 편하게 연락해 주세요.
